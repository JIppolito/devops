This plugin will send a single instance build request to Kitchen and poll kitchen every 10 seconds for a response. If the response from kitchen is FAILED, but it still returns an IP Address then the plugin will change the build status to UNSTABLE.
The plugin will only label the build as FAILURE when:

  • Kitchen status response states that the build failed AND there is no IP Address associated
    • with the build.
  • Kitchen status response fails to return an instance id

All of the Inputs for this plugin must be a valid Parameterized String in the format $variable_name or ${variable_name}, else it will treat the input as a string.
It will NOT parse a combination of parameterized values and strings (i.e. $role_SomePlainText_$BUILD_NUMBER)

Sample HTTP Post Parameters String that is sent to kitchen from this plugin:
http://kitchen.devops.enernoc.net/node_deploy_post
{ role=ids-api&environment_single=stg&branch=master&ami=ami-a9f5bfc0&bootstrap=ubuntu-12.04&zone=us-east-1a&subnet=subnet-22296e0a&type=c1.medium&security=sg-24ea2b41&qty=1&cloud_tag_single=master_jenkins_164 }
